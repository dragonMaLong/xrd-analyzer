# 不只算一个 Scherrer 平均值：我开源了一个 XRD 晶粒尺寸分布分析工具

做纳米材料、催化剂或者粉体样品时，很多人都会用 XRD 来估算晶粒尺寸。最常见的方法是 Scherrer 方程：从峰宽算出一个平均晶粒尺寸。

这个方法很经典，也很方便。但它有一个天然限制：最后往往只给你一个数。

一个数有时候够用，但很多时候不够。尤其是做 Pt/C 催化剂、纳米氧化物、合金颗粒或者经过热处理的粉体材料时，我们真正关心的并不只是“平均晶粒有多大”，而是：

这个样品里的晶粒尺寸是集中在一个范围内，还是有宽分布？

有没有小晶粒和大晶粒同时存在？

不同衍射峰给出的晶畴信息是否一致？

热处理、老化、循环测试之后，尺寸分布到底怎么变了？

过去我写过一次类似的工具介绍，但当时没有把代码开源。这次我把工具整理成了一个公开仓库，名字叫 **XRDAnalyzer**，希望对同样需要处理 XRD 峰形和晶粒尺寸分布的朋友有一点帮助。

项目地址：
https://github.com/dragonMaLong/xrd-analyzer

【配图 1：封面图，建议用“XRD 峰形 -> 晶粒尺寸分布”的示意图】

## 这个工具想解决什么问题

XRD 峰展宽中包含着晶粒尺寸的信息。晶粒越小，衍射峰通常越宽；晶粒越大，峰通常越窄。Scherrer 方程就是利用这个关系，把峰宽转换成一个平均晶粒尺寸。

但真实样品很少只有一个尺寸。一个样品中可能同时存在 2 nm、5 nm、10 nm 甚至更大的晶粒。如果我们只用一个平均值来描述，就会把这些信息压扁。

XRDAnalyzer 的目标不是再给一个平均值，而是尝试从 XRD 衍射峰形中恢复出一条 **晶粒尺寸分布**。

换句话说，它想回答的问题是：

不是“这个样品晶粒平均多大”，而是“这个样品的晶粒尺寸大概怎么分布”。

【配图 2：左侧为 Scherrer 平均值，右侧为晶粒尺寸分布曲线】

## 它不是替代 TEM、DLS 或 SAXS

这里需要先说明一点：这个工具并不是用来替代 TEM、DLS、激光粒度或 SAXS。

这些方法看的对象不完全一样。

TEM 看到的是投影颗粒尺寸和形貌；DLS 更偏向溶液中水合动力学粒径，而且对大颗粒或团聚体非常敏感；SAXS 反映的是纳米尺度散射结构，常常还需要形状模型；激光粒度更适合微米级颗粒或团聚体。

XRD 对相干衍射晶畴更敏感。对于负载型催化剂或者纳米晶材料，颗粒尺寸、团聚体尺寸和晶粒尺寸并不总是同一个概念。

所以我更愿意把 XRDAnalyzer 看作一个补充工具：当你已经有 XRD 数据，并且希望从峰展宽里挖出更多晶粒尺寸信息时，它可以提供一种新的分析入口。

## 核心思路：把峰形分析变成反问题

这个工具背后的想法可以这样理解：

如果一个样品中只有某一个固定尺寸的晶粒，那么它会贡献一种特定宽度和形状的衍射峰。

如果样品中有很多不同尺寸的晶粒，那么实验中看到的衍射峰，就可以看作是许多不同尺寸晶粒贡献的叠加。

于是问题就变成了：

已知最终看到的 XRD 峰形，能不能反过来推测每个晶粒尺寸贡献了多少？

这就是一个典型的反问题。

在代码里，我用 Pearson VII 函数描述不同晶粒尺寸对应的峰形核函数，用 Scherrer 关系把晶粒尺寸和峰宽联系起来，再把整个问题离散化成矩阵形式。

最后求解时使用：

- 非负最小二乘法 NNLS，保证得到的尺寸分布不出现负值；
- Tikhonov 正则化，让反演结果更稳定，不至于被噪声带着乱跑；
- L-Curve 方法，帮助选择合适的正则化参数 α。

这个方法的优点是：不用预先假设晶粒尺寸一定是正态分布、对数正态分布或者某个固定函数形式。它直接从峰形数据里反演分布。

【配图 3：流程图，XRD 峰形 -> 峰形基矩阵 -> NNLS + Tikhonov -> 晶粒尺寸分布】

## 目前软件能做什么

现在的 XRDAnalyzer 主要是一个带图形界面的 Python 工具。

它支持的功能包括：

- 多峰联合拟合：可以同时处理多个衍射峰，例如 Pt 的 111、200、220、311 等峰。
- Kα1/Kα2 双线处理：支持 Cu、Co、Fe、Mo 靶，自动计算 Kα2 的峰位贡献。
- 正则化反演：通过 NNLS + Tikhonov 正则化求解晶粒尺寸分布。
- L-Curve 分析：自动扫描一组 α 值，给出推荐的平滑参数。
- 仪器展宽入口：可以输入仪器本身的 FWHM，用来减弱仪器展宽对结果的影响。
- 数据读取：支持两列 TXT，也尝试支持 Bruker RAW、Rigaku RAW/Ultima 等格式。
- 图形交互：可以拖动峰位和分析范围，点击图例显示或隐藏不同峰的分布曲线。
- CSV 导出：可以导出粒径分布、原始数据、背景、总拟合和各峰贡献，方便后续用 Origin 或 Excel 重新作图。

## 最简单的使用流程

如果你第一次使用，我建议不要一上来就把所有峰都勾上。

最稳妥的方式是先用一个干净的峰把流程跑通。

基本步骤如下：

1. 用 JADE 或其他软件把原始 XRD 数据导出成两列 TXT。
2. 第一行写样品名，后面每一行是 2θ 和强度。
3. 打开 XRDAnalyzer，点击“导入文件”。
4. 用蓝色范围线框住一个清楚的衍射峰，并保留一点背景。
5. 选择 X 射线源，例如常见的 Cu 靶。
6. 设置晶粒尺寸范围，例如先用 0.5 nm 到 100 nm。
7. 平滑因子 α 可以先用默认值 1，不确定时再跑 L-Curve。
8. 先点“极速计算”看拟合是否正常，再点“精细计算”。
9. 检查拟合曲线是否能跟住原始峰。
10. 最后点击“保存结果”导出 CSV。

## 结果应该怎么看

计算结束后，最重要的是看两类图。

第一类是 XRD 拟合图。它告诉你模型能不能合理地跟住原始衍射峰。如果拟合曲线和原始数据明显对不上，那么后面的晶粒尺寸分布就不能盲目相信。

第二类是晶粒尺寸分布图。横坐标是晶粒尺寸，单位是 nm；纵坐标是相对分布强度。黑色虚线通常代表总分布，彩色曲线对应不同衍射峰给出的分布。

如果多个峰给出的分布趋势比较一致，结果通常更有说服力。如果不同峰之间差别很大，就需要回头检查峰选择、背景、重叠峰、仪器展宽，或者考虑是否存在微应变、缺陷、织构等因素。

这个工具不会替你判断材料体系是否满足所有假设。它提供的是一个可交互、可复现的反演框架，最终解释仍然需要结合样品本身和其他表征结果。

## 一些使用边界

我不建议把这个工具当成“导入数据后一键得到真实粒径”的黑箱。

它更适合作为一个分析辅助工具。

使用时尤其要注意：

- 峰的信噪比要尽量好。
- 分析范围不要混入明显重叠的其他峰。
- 背景选择会影响反演结果。
- 仪器展宽如果能用标准样品校正，结果会更稳。
- 如果峰展宽主要来自微应变、缺陷或其他效应，只用尺寸模型解释就可能偏差很大。
- RAW 文件格式受仪器厂家和软件版本影响很大，最稳的输入方式仍然是导出成规范的两列 TXT。

简单说，XRDAnalyzer 能帮你从峰形里提取更多信息，但它不能替代材料分析中的物理判断。

## 为什么开源
把这个工具开源，主要有三个原因。
第一，我希望它能被更多有类似需求的人试用。很多时候实验数据已经测了，只是缺一个顺手的小工具把峰形信息进一步挖出来。
第二，XRD 峰形反演本身有很多可以继续改进的地方。比如更好的仪器函数、更完整的微应变模型、更稳健的背景处理、更严格的多峰约束等，这些都适合在开源基础上继续讨论。
第三，我也希望这个项目能作为一个例子：科研里的很多小工具，不一定非要做成很复杂的软件。只要把方法、代码和边界讲清楚，就能帮助别人复现、修改和使用。

## 如何获取

GitHub 地址：
https://github.com/dragonMaLong/xrd-analyzer

Windows 打包版下载：
https://github.com/dragonMaLong/xrd-analyzer/releases/tag/v0.1.0

直接下载：
https://github.com/dragonMaLong/xrd-analyzer/releases/download/v0.1.0/XRDAnalyzer_MKL_win64.zip

下载后请先完整解压，再进入文件夹双击 `XRDAnalyzer.exe`。不要只把 `XRDAnalyzer.exe` 单独拿出来，旁边的 `_internal` 文件夹也必须保留。

SHA256：
`43CFC954CFB62DFD0843BC904B335D8F32889992CBB39C74D79D107C3C45B6E1`

源码运行方式：

```bash
git clone https://github.com/dragonMaLong/xrd-analyzer.git
cd xrd-analyzer
conda env create -f environment.yml
conda activate xrd-analyzer
python run.py
```

如果不使用 conda，也可以尝试：

```bash
pip install -r requirements.txt
python run.py
```

由于程序里包含较多矩阵运算，推荐优先使用 conda 环境。NumPy/SciPy 如果链接到 MKL 或高性能 OpenBLAS，精细计算和 L-Curve 扫描会更快。

## 最后

这个工具目前仍然是面向科研数据分析场景的小工具，不是一个可以无脑套用所有 XRD 图谱的万能软件。

但如果你手里有纳米晶材料、Pt/C 催化剂或者其他峰展宽明显的 XRD 数据，并且不满足于只得到一个 Scherrer 平均值，欢迎试试这个项目。

也欢迎在 GitHub 上提 issue、给建议，或者直接基于代码做你自己的修改。

我会继续把这个工具往更稳定、更清楚、更容易复现的方向整理。
